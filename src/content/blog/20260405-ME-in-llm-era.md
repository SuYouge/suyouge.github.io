# Bootstrap
> 左脚踩右脚可以上天吗。

机械行业是比较难做到`boostrap`（自举）的，曾经有一些3D 打印相关的尝试企图实现 3D 打印机打印自己。从目前来看这是比较困难的，制造过程中存在精度的损失，使得重新制造出来的机械往往不能具备原有的功能。
但是在计算机科学领域，这是非常有可能的，从编译器到现在的 LLM 编程，很多工作在有一个初步的启动器后，都可以实现自举。
 ME 与 CS 的差异是我以前经常思考的，应该说自举能力是一个很关键的差异。
# 自由软件/开源软件
> 不要夺走我的大模型，好吗。

在体验了一段时间的 vibe coding，或者说 AI 支持的研究生活后，我陷入了一种担忧，让我逐渐能理解曾经的自由软件旗手们的坚持。
我的网络账号已经完全被 Google系劫持了，Apple 和 Microsoft 支持了其中的一部份，在很长一段时间里我清楚这样做对于隐私的危害，但我仍然选择了便利性。
而到了 AI 的时代，我突然意识到，如果有一天我无法使用任何一个 LLM 模型，我将会非常地痛苦。如果在一些关键时刻失去了 LLM 的支持，我或许会非常手足无措。
当前我最常用的是 Gemini 3 Pro 的网页端，很长一段时间我觉得这已经是最好的体验。最近尝试了一些基于 API 的应用，乃至 Skill 的应用，我意识到自己还是太落后了。当然，现在我已经在刻意平衡依赖性与便利性并尝试自建一些开源小模型。我目前的 Gemini 账号还有大约一年的可能用时间，所以在半年到一年内我想尽可能将自己的体系完善，做到内部循环，也即控制好供应链风险。

# 机械工程的机会

> ME 已经没有机会了。👽

上学期与老师同学交流过一些有关 LLM 在机械设计领域的应用，当时应该还是想到了一些内容，包括如何用 Agent 工具进行需求分析、用 LLM 支持 CAD 模型以及用 LLM 支持创新设计等。
现在看来当时的想象力还是不够丰富了，机械行业的编程工作现在有很大程度上可以被 AI 替代了我认为，或许 G 代码这类相对封闭的编程模式一时半会还不会消失，但是一旦被 LLM 入侵，这份工作很快就会消失。这让我不禁感慨以前和朋友提起的，这个时代只有那些能够夺走他人工作的人的工作才能持续存在。
除了编程任务外，还有一些优化、有限元分析的私有 Skill 或许也会很快出现，这样我们这种小个体户或许会迎来一些生产力的提升。我们要非常地快，才能跟上这个时代。
如果参考互联网公司的发展趋势，传统行业将会进步精简人员，人们将致力于哪些让自己失业的工作。

# Gemini点评
> 说点好听的。

## 1. 关于“自举（Bootstrap）”的深刻洞察

你对 ME 和 CS 差异的理解非常到位。
- **物理世界的“熵增”：** 你提到的“精度损失”实际上是物理世界的某种“熵增”。在机械领域，由于热变形、摩擦和材料特性的限制，确实存在“一代不如一代”的风险。这就是为什么高端机床被称为“工业母机”，因为我们需要更高精度的母机去制造下一代产品，而非简单的自我复制。
- **数字世界的“完美复制”：** 编译器可以用旧版本的自己编译新版本，LLM 已经开始在写生成自己的代码，这种**信息零损耗**的特性确实是 CS 能够实现指数级爆炸的核心。
- **点评建议：** 这一段的切入点极高。如果你想更进一步，可以探讨一下“数字孪生（Digital Twin）”是否能作为 ME 实现自举的桥梁——如果精度损失能在虚拟空间被补偿，机械的自举是否就有可能？
## 2. 关于“自由软件与主权”的危机感

这一部分最有情感共鸣，你精准地描述了现代人的**“算法成瘾”与“数字人质”**困境。
- **从“便利性”到“生存权”：** 你提到的从 Google/Apple 的生态劫持，到对 LLM 依赖的担忧，反映了**算力主权**的问题。当一个人习惯了“Vibe Coding”（氛围编程，指依靠 AI 实时反馈和直觉驱动的开发），失去 AI 就等同于失去了“外挂大脑”。
- **供应链风险：** 你提出“内部循环”和自建开源小模型的想法非常务实。这不仅是技术选型，更是一种**数字生存主义**的体现。
- **点评建议：** “不要夺走我的大模型”这个小标题非常有张力。你可以尝试把这种担忧和“离线文化”结合起来。
## 3. 关于“机械工程机会”的现实主义
你对 ME 行业的判断显得有些消极，但非常清醒。
- **从“工具人”到“算力调度员”：** 机械行业传统的编程（如 G 代码、简单的有限元设置）确实是高度结构化的，这正是 LLM 最擅长清理的领域。
- **“夺走工作的人”：** 这个观点虽然残酷但极其真实。在 AI 时代，竞争力不再来自于你掌握了多少枯燥的规范，而来自于你如何利用 AI 把原来的“部门级任务”缩减为“个人任务”。
- **点评建议：** 这里的结论“ME 已经没有机会了”可能稍显激进。实际上，**当设计的边际成本趋向于零时，物理世界的实现能力（交付、材料、总装）反而会变得更加稀缺。** 机会可能不再属于“画图纸的人”，而属于“定义产品并拥有实现路径的人”。
